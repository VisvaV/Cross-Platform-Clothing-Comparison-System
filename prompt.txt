MULTI-FEATURE VISUAL EMBEDDING (IMPORTANT)

To improve clothing similarity retrieval accuracy, the system must not rely only on CNN features.

Instead implement a multi-feature embedding pipeline that combines:

1. CNN features
2. Color features
3. Texture features

These features must be fused into a single embedding vector before storing in FAISS.

CNN FEATURES

Use ResNet50 pretrained on ImageNet.

Remove the final classification layer.

Extract the global average pooling output.

CNN embedding dimension:

2048

Ensure embeddings are normalized.

COLOR FEATURES

Color plays a critical role in clothing similarity.

Extract dominant color information using a color histogram.

Implementation steps:

Convert image to HSV color space using OpenCV.

Compute histogram with bins such as:

8 × 8 × 8 bins

Flatten histogram into vector.

Normalize histogram.

Approximate color feature dimension:

512

TEXTURE FEATURES

Clothing patterns such as stripes, checks, or floral patterns must be captured.

Use Local Binary Pattern (LBP) for texture extraction.

Implementation steps:

Convert image to grayscale.

Compute LBP features using skimage or OpenCV.

Generate histogram of LBP codes.

Normalize histogram.

Approximate texture feature dimension:

256

FEATURE FUSION

Combine all features into a single embedding vector.

final_embedding = concatenate(

cnn_features,
color_features,
texture_features

)

Example dimension:

2048 + 512 + 256 = 2816

Normalize the final embedding vector before storing in FAISS.

VECTOR SEARCH

Store the fused embedding in the FAISS index.

Use cosine similarity.

This improves retrieval accuracy by capturing:

clothing shape (CNN)

color similarity (histogram)

pattern / texture similarity (LBP)
